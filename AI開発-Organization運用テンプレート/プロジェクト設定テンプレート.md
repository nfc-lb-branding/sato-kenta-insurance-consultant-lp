## 作業時の注意事項

### リポジトリ操作
- 新規リポジトリ作成時は必ず Organization 配下に作成
- push 前に `git remote -v` で URL が正しいことを確認
- 403 エラーが出た場合は Windows 資格情報マネージャーで GitHub 資格情報を削除し、PAT で再認証

### コミット規約
- コンベンショナルコミット形式を使用: `feat:`, `fix:`, `docs:`, `chore:` など
- 変更内容は簡潔かつ明確に記述
